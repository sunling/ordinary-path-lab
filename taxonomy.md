# 咨询资料分类规范

这份文件定义 `咨询资料整理` 系统里的字段、主题、标签和隐私规则。它服务于搜索、统计和复用，不是为了把人固定在某个分类里。

如果一条记录横跨多个主题，`primary_theme` 只放最主要的一个，其他主题用 `tags`、`layers` 和正文里的“可进入的主题聚合”补充。

## 文件层级 `kind`

`kind` 由文件所在目录自动决定，不需要手动写入 frontmatter。

- `02-normalized`：标准化咨询记录。
- `03-issue-units`：单个可复用问题。
- `04-topic-clusters`：主题聚合页。
- `05-outputs`：可用化输出，如内容选题、服务雏形、连接池候选。
- `06-article-drafts`：从 issue/topic 发展出的文章草稿。
- `07-people`：咨询对象和联系方式管理。默认私密，不进入通用 records 索引。

## 资料类型 `record_type`

用于说明记录本身是什么类型。

- `qa_thread`：文字问答、留言、私信。
- `consult_call`：一对一咨询转写或会议记录。
- `coaching_session`：人生教练、连续陪伴或会后记录。
- `collaboration`：合作邀约、采访邀约、商业沟通。
- `boundary`：加微信、沟通渠道、私域边界。
- `output`：内容选题、服务雏形、连接池等可用化输出。
- `article_draft`：文章草稿。
- `other`：暂未归类。

## 整理状态 `status`

- `raw`：只有原始材料。
- `indexed`：已登记来源。
- `normalized`：已整理成标准化咨询记录。
- `split`：已拆出问题单元。
- `clustered`：已进入主题聚合。
- `reusable`：已可复用为 FAQ、文章、服务或连接池素材。

## 隐私等级 `privacy_level`

默认使用 `private`。只有经过明确处理后，才使用更开放的等级。

- `private`：默认值，只在本地使用。
- `anonymized`：已去除姓名、联系方式、具体公司、具体学校等敏感信息。
- `public_ready`：可以作为公开内容素材，但发布前仍需复查。

### 必须保留为 `private` 的情况

- 债务、借钱、关系财务纠纷。
- 家暴、抑郁、亲人离世、严重创伤。
- 签证身份、灰色路径、法律风险。
- 商业合作细节、报价、未公开项目。
- 具体公司、学校、收入、城市流动组合后容易识别本人。

### 可用 `anonymized` 的情况

- 已删除姓名、微信、邮箱、电话。
- 已模糊具体城市、公司、学校、项目名。
- 已删除债务、家庭创伤、签证身份等高度敏感细节。
- 保留的是处境结构，而不是可识别个人。

### `public_ready` 使用条件

必须同时满足：

- 不含可识别身份信息。
- 不含高度敏感私人事件。
- 不会让当事人被熟人识别。
- 涉及政策、签证、学校申请、学费、就业市场时，已准备发布前查官方来源。

## 三层生态 `layers`

`layers` 标记一条记录未来可能进入哪一层。可以多选。

- `content`：适合沉淀成 FAQ、文章、案例、选题。
- `service`：适合深聊、路径诊断、一对一咨询、小圆桌。
- `connection`：适合连接相似处境的人。

示例：

```yaml
layers:
  - content
  - service
  - connection
```

## 连接池判断

### 适合进入 `connection`

- 处境相似：小城市女性、转码、海外求职、Day 1 CPT、非典型学历路径等。
- 资源互补：有人有经验，有人有问题，有人能提供路径或行业信息。
- 情绪互助：不是单纯知识问题，而是需要知道“我不是一个人”。
- 风险较低：即使进入小圆桌，也不会暴露高度敏感隐私。

### 不适合进入 `connection`

- 对方只是一次性索取答案。
- 资料含高度敏感隐私，且未获得分享授权。
- 问题更适合一对一深聊，不适合群体环境。
- 债务、借钱、家暴、抑郁、亲人离世、签证身份、商业机密。
- 对方尚未表达愿意被连接。

## 命名规则

### normalized record

建议格式：

```text
consult-YYYYMMDD-person-topic.md
```

如果只有月份：

```text
consult-YYYYMM-topic.md
```

示例：

```text
consult-20260308-kerwin-us-master-cloud-computing.md
consult-202211-career-vs-grad-school-family-opinion.md
```

### issue unit

建议格式：

```text
issue-YYYYMMDD-slug.md
```

示例：

```text
issue-20260606-liberal-arts-cs-fit.md
issue-20260606-overseas-not-fast-debt-solution.md
```

### topic cluster

建议格式：

```text
topic-slug.md
```

示例：

```text
topic-overseas-study-work-path.md
topic-career-start-rebuild.md
```

### output

建议使用功能名：

```text
content-backlog.md
service-offers.md
connection-pool.md
```

### article draft

建议格式：

```text
article-YYYYMMDD-slug.md
```

示例：

```text
article-20260606-liberal-arts-cs-fit.md
```

## 发布前检查

从本系统生成公开内容前，至少检查：

- 是否删除姓名、微信、邮箱、手机号。
- 是否模糊城市、学校、公司、收入等组合识别信息。
- 是否删除债务、家暴、抑郁、亲人离世、签证身份等敏感细节。
- 是否把个人故事写成了可复制承诺。
- 是否把过来人经验写成了官方政策。
- 涉及学校、签证、学费、就业市场的信息是否需要重新查证。
